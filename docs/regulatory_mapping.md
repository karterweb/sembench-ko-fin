# Regulatory Mapping

This benchmark is aligned with the Korean `가명정보 처리 가이드라인(2026.03.)` as a technical test harness.

| Guideline concern | Benchmark dimension |
|---|---|
| Processing purpose and necessity | SPS intent/actionability dimensions |
| Utility preservation | SPS fact/reasoning dimensions |
| Re-identification risk | ComplianceChecker and L3 cross-reference cases |
| Additional-information separation | Adapter contract and L4 regulatory cases |
| Suitability review | HTML report and per-case violations |
| Residual risk monitoring | L3/L4 risk tiers and validation workflow |

This project is not a certification. It gives AI, data, and security teams evidence to discuss before Innovative Financial Services applications or production operation.
